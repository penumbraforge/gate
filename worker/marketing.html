.env committed to git

A developer pushes database credentials to a public repo. Automated scrapers find it in minutes. Your production database is compromised before the PR is reviewed.

API key in source code

A Stripe live key hardcoded in a config file. It survives code review, CI passes, deploys to production. Three months later, fraudulent charges appear.

SSH key in Docker image

Private keys baked into container layers persist in registry history forever. Rotating the key doesn't help if the old image is cached across 40 nodes.

Secrets in CI logs

Environment variables echoed during debug builds. Log aggregation indexes them. Now your secrets live in Elasticsearch with a 90-day retention you forgot about.

Encryption

Vault

AES-256-GCM local encryption. Secrets are encrypted on your machine and never transmitted. Use the VAULT: prefix for automatic decryption at runtime.

How it works

Generate a local key, encrypt your secrets, and commit the encrypted values safely. At runtime, Gate decrypts VAULT:-prefixed values automatically.

  • AES-256-GCM authenticated encryption
  • Keys generated and stored locally
  • Encrypt individual values or entire .env files
  • VAULT: prefix for automatic decryption
  • Safe to commit encrypted values to git
  • Zero network dependency
vault — encrypt
# Generate a vault key
$ gate vault keygen
▸ Key saved to .gate/vault.key
▸ Added .gate/vault.key to .gitignore

# Encrypt a secret value
$ gate vault encrypt "sk_live_4eC39Hq..."
VAULT:v1:aGVsbG8gd29ybGQ=:abc123...

# Encrypt entire .env file
$ gate vault env .env

Trust model

Zero-trust by design.

Gate doesn't ask for access to your code, your repos, or your infrastructure. Everything runs locally.

Local-Only Scanning

All scanning happens on your machine. No code is ever transmitted to external servers. No telemetry by default.

Keys Never Leave

Vault encryption keys are generated and stored locally. Encrypted values can be committed safely — only your machine can decrypt them.

Tamper-Proof Rules

Detection rules are HMAC-SHA256 signed. Any modification is detected and rejected. Reproducible scans across every environment.

Full Audit Trail

Every scan, bypass, and remediation is logged with SHA-256 integrity hashes. Filterable, exportable, tamper-evident.

No Vendor Lock-in

Self-hosted dashboard. Portable rules. Open source CLI. Switch away at any time without losing your data.

Compliance Ready

Architecture aligns with OWASP secret management guidelines, NIST SP 800-53 controls, and CIS benchmark recommendations.
