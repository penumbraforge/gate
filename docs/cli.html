CLI Reference

All Gate commands and options.

Commands

gate scan                    # Scan files for secrets
gate serve                   # Start API + dashboard on :3000
gate serve --port 8080       # Custom port
gate worker                  # Start background job processors
gate install                 # Install pre-commit hook
gate audit --since 7d        # Query audit log
gate auth GATE-PRO-TEAM-...  # Activate license
gate version                 # Show version
gate help                    # Show all commands

gate scan

Scans files for secrets using the FORTRESS rule engine (50+ patterns across 26 categories). Pure JavaScript, no build or database required.

# Scan staged files (pre-commit mode)
gate scan

# Scan specific files
gate scan config.js .env secrets.yaml

# Scan with JSON output
gate scan --format json

Detection Pipeline

  1. File Input — reads file content line by line
  2. Pattern Matching — tests each line against 50+ regex rules
  3. Entropy Analysis — calculates Shannon entropy on suspicious tokens (threshold: 3.8 bits/char)
  4. False Positive Filtering — skips known safe patterns (test fixtures, example values)
  5. Result Aggregation — groups findings by file and severity (critical/high/medium/low)

gate install

Installs a Git pre-commit hook that runs gate scan on staged files before every commit.

gate install

The hook shows interactive prompts when secrets are found:

Found potential AWS secret in config.js
(f)ix it now | (s)kip | (b)ypass | (c)ancel

gate serve

Starts the Express API server and React dashboard on a single port.

# Default (port 3000)
gate serve

# Custom port
gate serve --port 8080

# API only (no dashboard)
DISABLE_DASHBOARD=1 gate serve
OptionDefaultDescription
--port3000Port to listen on
DISABLE_DASHBOARDunsetSet to 1 for API-only mode

Requires a build step first (npm run build) and PostgreSQL.

gate audit

Queries the local audit log stored at ~/.gate/audit.jsonl.

gate audit --since 7d          # Last 7 days
gate audit --stats             # Aggregate statistics
gate audit --export json       # Export full log
gate audit --verify            # Verify integrity chain

Each entry includes timestamp, commit hash, files scanned, findings with severity, user decision, and a SHA-256 integrity hash chain.

gate auth

Manages license activation and credentials.

gate auth GATE-PRO-TEAM123-2027-01-01-abc123    # Activate license
gate license                                      # Show license status
gate logout                                       # Clear credentials

License keys encode: plan type, team ID, expiry date, and HMAC signature. Free plans get 50 scans/month.

gate vault

Manages local AES-256-GCM encryption for secrets. The vault key is 32 random bytes stored as hex in ~/.gate/vault.key (chmod 0600).

gate vault keygen              # Generate ~/.gate/vault.key
gate vault encrypt <value>     # AES-256-GCM encrypt → base64 blob
gate vault decrypt <blob>      # Decrypt vault blob → plaintext
gate vault env <file>          # Encrypt all .env values → .env.encrypted

Output Format

Encrypted values are base64-encoded JSON containing: algo, iv, ct (ciphertext), and authTag. A random IV is generated per encryption operation.

VAULT: Prefix

Environment variables prefixed with VAULT: are automatically decrypted at server startup using the vault key. Example:

# .env.encrypted
DATABASE_URL=VAULT:eyJhbGdvIjoiYWVzLTI1Ni1nY20iLC...
REDIS_URL=VAULT:eyJhbGdvIjoiYWVzLTI1Ni1nY20iLC...

See Vault & Encryption for full details.

gate setup

One-command project setup that handles dependencies, database, Prisma client generation, build, and Git hook installation.

gate setup                     # Full project setup (deps, db, prisma, build, hook)
gate setup --skip-db           # CLI-only install (no PostgreSQL)

gate setup is idempotent — safe to run multiple times. It skips steps that are already complete.

Remediation Guidance

Scan output now includes grouped remediation guidance for each finding. Each finding is annotated with an action and links to provider documentation.

ActionMeaning
ROTATESecret is exposed — rotate immediately via provider dashboard
MOVE TO ENVHardcoded value should be moved to an environment variable
ENCRYPTSensitive value should be encrypted using gate vault encrypt
USE VAULTUse VAULT: prefix for automatic decryption at startup
REVIEWManual review needed — may be a false positive or test fixture

Each finding also links to the relevant provider's documentation for credential rotation (e.g., AWS IAM console, GitHub token settings, Stripe API key management).
