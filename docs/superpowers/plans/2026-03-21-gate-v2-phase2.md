# Gate v2 Phase 2: Remediation Engine

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the remediation engine that makes Gate the first secret scanner that doesn't just detect secrets but verifies them, fixes them, and documents the entire incident response — for free.

**Architecture:** Seven new modules added to `src/cli/`, each with a single responsibility. The fixer rewrites code using pattern-based replacement (no AST dependency). The verifier makes read-only HTTP calls to provider APIs. The interactive mode uses raw readline for single-keypress input. All modules are pure JS, no build step.

**Tech Stack:** Node.js (>=18), Jest for testing, Node built-in `https`/`http` for verification, `readline` for interactive TUI, `crypto` for snapshot hashing

## Task 1: Exposure Assessment Module

**Files:**
- Create: `src/cli/exposure.js`
- Test: `src/cli/__tests__/exposure.test.js`

### What it does

Determines how exposed a secret is based on git state:
- **LOCAL** — file is staged but not committed (safe — just extract it)
- **COMMITTED** — in local commits but not pushed (extract + amend)
- **PUSHED** — in remote-tracking branches (full incident response needed)
- **UNKNOWN** — can't determine (treat as potentially exposed)

### Implementation

```js
// src/cli/exposure.js
const { execSync } = require('child_process');

/**
 * Assess the exposure level of a file containing a secret.
 * @param {string} filePath — path relative to repo root
 * @param {string} [cwd] — working directory
 * @returns {{ level: string, confidence: string, details: string }}
 */
function assessExposure(filePath, cwd) {
  cwd = cwd || process.cwd();
  const opts = { encoding: 'utf8', cwd, stdio: ['pipe', 'pipe', 'ignore'] };

  try {
    // Is the file only staged (not committed)?
    const staged = git('diff --cached --name-only', opts);
    if (staged.split('\n').some(f => f.trim() === filePath)) {
      // Check if it's also in HEAD
      try {
        const inHead = git(`cat-file -e HEAD:${filePath}`, opts);
        // File exists in HEAD — check if the staged version has changes
      } catch {
        // File does NOT exist in HEAD — it's a new staged file
        return { level: 'LOCAL', confidence: 'high', details: 'Staged but never committed' };
      }
    }

    // Is the file in any commit?
    const commits = git(`log --all --oneline -- "${filePath}"`, opts);
    if (!commits.trim()) {
      return { level: 'LOCAL', confidence: 'high', details: 'Not in any commit' };
    }

    // Is it in remote-tracking branches?
    const remoteCommits = git(`log --remotes --oneline -- "${filePath}"`, opts);
    if (remoteCommits.trim()) {
      // Find how long it's been exposed
      const firstRemote = git(`log --remotes --diff-filter=A --format=%aI -- "${filePath}"`, opts).trim().split('\n').pop();
      const days = firstRemote ? Math.ceil((Date.now() - new Date(firstRemote).getTime()) / 86400000) : null;
      return {
        level: 'PUSHED',
        confidence: 'based on local refs',
        details: days ? `In remote history for ~${days} days` : 'In remote history',
      };
    }

    // In local commits only
    return { level: 'COMMITTED', confidence: 'high', details: 'Committed locally but not pushed' };
  } catch {
    return { level: 'UNKNOWN', confidence: 'low', details: 'Could not determine exposure — treat as potentially exposed' };
  }
}

function git(cmd, opts) {
  return execSync(`git ${cmd}`, opts);
}

/**
 * Format exposure level for terminal display.
 */
function formatExposure(exposure, useColor) {
  const labels = {
    LOCAL: 'LOCAL ONLY',
    COMMITTED: 'COMMITTED — not pushed',
    PUSHED: 'COMPROMISED',
    UNKNOWN: 'UNKNOWN',
  };
  const label = labels[exposure.level] || exposure.level;
  // Color: LOCAL=green, COMMITTED=yellow, PUSHED=red, UNKNOWN=dim
  return { label, details: exposure.details, confidence: exposure.confidence };
}

module.exports = { assessExposure, formatExposure };
```

## Task 2: Credential Verification Module

**Files:**
- Create: `src/cli/verify.js`
- Test: `src/cli/__tests__/verify.test.js`

### What it does

Makes read-only API calls to provider endpoints to check if a detected credential is actually live. Uses Node built-in `https`/`http` modules (no axios dependency).

### Key design decisions
- 2-second timeout per provider
- 5-second total budget across all verifications
- All calls run in parallel via `Promise.allSettled`
- Cache results in `~/.gate/verify-cache.json` with 1-hour TTL
- Disabled by default in pre-commit hook, enabled in `gate scan`
- `--no-verify` flag or `verify: false` in `.gaterc` disables globally
- Never sends credentials to third parties — only to the provider's own API

### Provider verification functions

Each provider has a `verify_<provider>(credential)` function that returns:
```js
{ status: 'live' | 'inactive' | 'error' | 'timeout', details: {} }
```

3. GitLab (`GET /api/v4/user` with Private-Token header)
4. Stripe (`GET /v1/balance` with Bearer token)
5. Slack (`POST auth.test` with Bearer token)
6. OpenAI (`GET /v1/models` with Bearer token)
7. Anthropic (`GET /v1/models` with x-api-key header)
8. GCP (OAuth2 tokeninfo endpoint)
9. Twilio (`GET /2010-04-01/Accounts/{sid}`)
10. SendGrid (`GET /v3/user/profile` with Bearer token)

## Task 3: Auto-Fix Engine (Core)

**Files:**
- Create: `src/cli/fixer.js`
- Test: `src/cli/__tests__/fixer.test.js`

### What it does

The auto-fix engine extracts secrets from source code into `.env` files, rewrites code to use environment variable references, and manages `.env`/`.gitignore` files. This is the heart of what makes Gate unique.

### Module exports

```js
module.exports = {
  fixFinding,        // Fix a single finding in a file
  fixAll,            // Fix all findings in scan results (batch mode)
  dryRun,            // Preview what fix would do without changing files
  undo,              // Revert the most recent fix
  deriveEnvVarName,  // Derive env var name from rule ID and code context
  detectLanguage,    // Detect language from file extension
  getRewriter,       // Get the pattern rewriter for a language
};
```

### Language-aware rewriting

Each language has a rewriter object:
```js
const REWRITERS = {
  javascript: {
    extensions: ['.js', '.jsx', '.mjs', '.cjs'],
    envRef: (name) => `process.env.${name}`,
    dotenvSetup: { require: "require('dotenv').config();", import: "import 'dotenv/config';" },
    detectDotenv: (content) => content.includes("require('dotenv')") || content.includes("import 'dotenv"),
  },
  typescript: {
    extensions: ['.ts', '.tsx'],
    envRef: (name) => `process.env.${name}`,
    dotenvSetup: { require: "require('dotenv').config();", import: "import 'dotenv/config';" },
    detectDotenv: (content) => content.includes("require('dotenv')") || content.includes("import 'dotenv"),
  },
  python: {
    extensions: ['.py'],
    envRef: (name) => `os.environ["${name}"]`,
    importLine: 'import os',
    detectImport: (content) => /^import os$/m.test(content),
  },
  go: {
    extensions: ['.go'],
    envRef: (name) => `os.Getenv("${name}")`,
    importPkg: '"os"',
  },
  ruby: {
    extensions: ['.rb'],
    envRef: (name) => `ENV["${name}"]`,
  },
  java: {
    extensions: ['.java'],
    envRef: (name) => `System.getenv("${name}")`,
  },
  yaml: {
    extensions: ['.yml', '.yaml'],
    envRef: (name) => `\${${name}}`,
  },
  dockerfile: {
    extensions: ['Dockerfile'],
    // Special handling: ENV X=val → ARG X
  },
  terraform: {
    extensions: ['.tf'],
    envRef: (name) => `var.${name.toLowerCase()}`,
    // Also generates a variable block: variable "name" { type = string }
  },
  json: {
    extensions: ['.json'],
    // JSON cannot reference env vars. Extract secret to .env,
    // leave a manual migration note explaining how to load from env.
    cannotInline: true,
  },
};
```

### Pattern-based rewriting

For each finding, the fixer:
1. Reads the file line
2. Identifies the secret match position
3. Replaces the secret value with the env var reference
4. Handles quoting: `"secret"` → `process.env.X` (removes quotes for JS), `'secret'` → `os.environ["X"]` (removes quotes for Python)

The core replacement logic:
```js
function rewriteLine(line, match, envRef) {
  // Find the secret in the line (including quotes)
  // Replace the quoted secret with the env var reference
  // Handle: "secret", 'secret', `secret`
}
```

### Env var name derivation (from spec)

Known mappings: `stripe-live-secret` → `STRIPE_SECRET_KEY`, `aws-access-key-id` → `AWS_ACCESS_KEY_ID`, etc.
Fallback: derive from variable name in code context (`const dbPassword =` → `DB_PASSWORD`).
Last resort: `SECRET_<N>`.

### .env handling

- Append to existing `.env` (never overwrite)
- Create `.env` if missing, with header comment
- Conflict handling: same value = no-op; different value = warn + suffix `_NEW`
- Add `.env` to `.gitignore` if not present
- Create `.env.example` with placeholders

### Snapshot for undo

Before making changes, save pre-fix state:
```
~/.gate/snapshots/<sha256-of-repo-path>/<timestamp>/
  manifest.json
  files/
    <original files>
```

### Tests

Cover ALL code rewriting patterns from the spec:
- JS: `const x = "secret"` → `process.env.X`
- JS: `{ key: "secret" }` → `{ key: process.env.X }`
- Python: `x = "secret"` → `os.environ["X"]` + import injection
- Go: `x := "secret"` → `os.Getenv("X")`
- Ruby: `x = "secret"` → `ENV["X"]`
- Java: `String x = "secret"` → `System.getenv("X")`
- YAML: `key: secret` → `key: ${X}`
- .env creation and appending
- .env conflict handling (same value, different value)
- .gitignore update (add .env, idempotent)
- Dotenv injection (JS: adds require, only once, detects ESM)
- Terraform: `secret = "value"` → `secret = var.name` + variable block generated
- JSON config: extracts to .env, provides manual migration note (cannot inline env refs)
- Dry-run mode (no files modified, output matches what fix would do)
- Undo (restores from snapshot, deletes gate-created files)
- Undo with no snapshots (helpful error message)
- Double undo (warns "no more snapshots to undo")
- Snapshot rotation (11th fix prunes oldest snapshot)
- ESM project (`"type": "module"` in package.json) uses `import 'dotenv/config'` not require
- Verification after fix (re-scan passes)
- Re-staging guardrails: only re-stage if re-scan finds zero findings
- Re-staging guardrails: do NOT re-stage if findings remain

    aws: 'aws', github: 'gh', stripe: 'stripe',
