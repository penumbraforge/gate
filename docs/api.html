API Routes

Complete reference for the Gate REST API.

Authentication

The API uses JWT tokens stored in httpOnly cookies. Authenticate via GitHub OAuth, then all subsequent requests include the cookie automatically.

OAuth Flow

  1. Redirect user to GET /auth/github
  2. GitHub redirects back to GET /auth/github/callback
  3. Backend creates JWT, sets auth_token cookie
  4. All API calls include the cookie automatically

Security

LayerProtection
TransportHTTPS (reverse proxy), HSTS via Helmet
AuthenticationGitHub OAuth + JWT tokens in httpOnly cookies
AuthorizationRole-based (owner/member/viewer) per team
CSRFDouble-submit cookie pattern on all mutations
Input validationZod validation on all request bodies
Secrets at restAES-256-GCM encryption for audit log content
Rate limitingPer-endpoint rate limits
HeadersHelmet sets X-Frame-Options, CSP, etc.
