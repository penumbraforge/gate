Dashboard

Team management, audit logs, and integrations.

The Gate dashboard is a React SPA served by the same Express server that runs the API. Login with GitHub, then manage everything from a single interface.

Stack

  • React 19 with TypeScript
  • Zustand for state management
  • TanStack Router for routing
  • Tailwind CSS v4 with custom Penumbra Forge theme
  • Vite 7 for building

Tabs

Overview

Scan stats, finding trends, and team health at a glance. Shows scan volume, detection rates, and recent activity.

Audit Log

Filterable log viewer with export to JSON/CSV. Search by date range, severity, file, or user decision. Click any finding to expand remediation guidance with rotate/encrypt/move-to-env actions and provider doc links.

Team

Invite/remove members with role management. Roles: owner, member, viewer.

License

License key display, copy, and regenerate. Shows plan type, expiry, and scan quota.

Integrations

Slack webhook setup and custom outgoing webhooks. Configure notifications for findings.

Billing

Current plan, upgrade/downgrade via Stripe, and invoice history.

Repositories

Connect GitHub repos, toggle scan-on-push per repository, and view scan history for each connected repository.

Scanner Config

Tune entropy threshold, enable/disable individual rules, manage allowlists for known-safe patterns, and set severity threshold for alerts.

Authentication

  1. User clicks "Login with GitHub"
  2. Redirects to GitHub OAuth (/auth/github)
  3. GitHub redirects back with code (/auth/github/callback)
  4. Backend creates/finds user in PostgreSQL, creates JWT
  5. JWT stored as auth_token cookie (httpOnly, secure, sameSite)
  6. All subsequent API calls include the cookie automatically

Running the Dashboard

Production

# Build frontend + backend
npm run build

# Start (serves dashboard + API on same port)
gate serve

Development

# Starts backend on :3000, frontend on :5173 with HMR
npm run dev

In dev mode, Vite proxies all API requests to localhost:3000. In production, the backend serves the frontend directly from dist/frontend/.

API-Only Mode

# Disable the dashboard, serve API only
DISABLE_DASHBOARD=1 gate serve
