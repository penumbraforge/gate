Quick Start

Get Gate running in under 2 minutes.

1. Install

# macOS / Linux
curl -fsSL https://gate.penumbraforge.com/install.sh | bash

# Windows (PowerShell)
irm https://gate.penumbraforge.com/install.ps1 | iex

# Or install manually
npm install -g @penumbra/gate

2. Set Up the Pre-Commit Hook

# Install the git hook
npx gate install

# That's it. Every commit is now scanned.
git commit -m "add config"
# Gate finds secrets, asks what to do

The pre-commit hook is pure JavaScript — no build step, no database, no external dependencies. It works immediately after install.

3. Add GitHub Actions (Optional)

For server-side enforcement that can't be bypassed:

# .github/workflows/gate.yml
name: Gate Security
on: [pull_request, push]
jobs:
  gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: penumbra/gate@v1
        with:
          mode: enforce
          slack-webhook: ${{ secrets.SLACK_WEBHOOK }}

See GitHub Action docs for all options.

4. Start the Dashboard (Optional)

For team management, audit logs, and integrations:

gate setup

gate setup handles everything automatically: installs dependencies, creates the database, generates the Prisma client, builds the frontend and backend, and installs the Git hook. Run it once and you're ready.

# Start the server
gate serve

Dashboard runs on http://localhost:3000 by default.

5. Encrypt Your Secrets (Optional)

# Generate a vault key
gate vault keygen
# => Key saved to .gate/vault.key
# => Added .gate/vault.key to .gitignore

# Encrypt a single secret value
gate vault encrypt "sk_live_4eC39HqLyjWD..."
# => VAULT:v1:aGVsbG8gd29ybGQ=:abc123...

# Encrypt your entire .env file
gate vault env .env
# => Encrypted 12 values
# => Written to .env.encrypted

Values prefixed with VAULT: are automatically decrypted at server startup. Your .env.encrypted file is safe to commit — only machines with the vault key can decrypt the values.

Vault key security: The vault key at .gate/vault.key is automatically added to .gitignore. Never commit this file. Share it securely with team members who need access (e.g., via a password manager or secure channel).

Using VAULT: prefix in .env files

# .env.encrypted (safe to commit)
STRIPE_KEY=VAULT:v1:aGVsbG8=:def456...
DB_PASSWORD=VAULT:v1:d29ybGQ=:ghi789...
APP_NAME=MyApp  # non-secret values stay plaintext

At server startup, Gate reads the vault key and decrypts any VAULT:-prefixed values in memory. Plaintext values pass through unchanged. See Vault & Encryption for full details.
