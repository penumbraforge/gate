Gate

The Defender of Your Repos — every commit through Gate, no secrets get past.

Gate is a zero-trust pre-commit security defender that stops secrets, sensitive data, and misconfigurations before they ever reach your repository.

Three-Layer Defense

1. Pre-Commit Hook

Scans staged files locally for 50+ secret patterns. Zero external dependencies, fully offline. Interactive prompts let you fix, skip, bypass, or cancel.

your machine

2. GitHub Actions

Runs on every PR/push. Blocks unsafe commits that can't be bypassed with --no-verify. Posts PR comments with findings.

server enforcement

3. Dashboard + API

Team management with role-based access. License key generation, audit log viewer, and integration setup (Slack, webhooks).

management

New in v2.1

Vault Encryption

AES-256-GCM local encryption. Encrypt individual values, entire .env files, or use the VAULT: prefix for automatic decryption at server startup. Secrets never leave your machine.

new

Remediation Guidance

Per-finding actionable advice with provider documentation links. Actions include: rotate credentials, move to environment variables, encrypt with vault. Every finding tells you exactly what to do.

Opt-in sanitized error collection for improving Gate. All reports are scrubbed of secrets and PII before transmission. Secrets never leave your infrastructure.

How Secrets Are Caught

Example: A developer commits an AWS key.

// config.js
const AWS_SECRET = "AKIAIOSFODNN7EXAMPLE";
  1. git commit → Pre-commit hook scans files locally
  2. Hook finds AWS secret pattern, shows interactive prompt
  3. Developer chooses to bypass → logged locally with timestamp
  4. git push → GitHub Actions runs server-side scan
  5. Action blocks the commit — can't bypass server enforcement
  6. Slack alert sent to security team
  7. Dashboard shows bypass attempt + full audit log

Result: Secret never made it to the repo. Bypass is recorded and visible.
