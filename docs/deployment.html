Deployment

Self-hosting Gate in production.

Prerequisites

  • PostgreSQL >= 14 (RDS, Supabase, Neon, etc.)
  • Redis >= 6 (ElastiCache, Upstash, etc.) — only if running workers
  • Node.js >= 18
  • GitHub OAuth App configured with your production callback URL

Environment Variables

In development, all variables have safe defaults. In production, you must set every variable below.

VariablePurpose
DATABASE_URLPostgreSQL connection string
REDIS_URLRedis connection (workers only)
JWT_SECRETJWT signing key (32+ chars)
ENCRYPTION_KEYAES-256 key (64 hex chars)
SESSION_SECRETExpress session secret (32+ chars)
GATE_LICENSE_SECRETLicense key HMAC signing (16+ chars)
GITHUB_CLIENT_IDGitHub OAuth app ID
GITHUB_CLIENT_SECRETGitHub OAuth app secret
GITHUB_CALLBACK_URLOAuth redirect URI
DASHBOARD_URLCORS origin for dashboard
NODE_ENVSet to production
GITHUB_WEBHOOK_SECRETGitHub webhook signing secret (required for webhook auto-scan)
ERROR_REPORTINGSet to true to enable opt-in error reporting (default: disabled)
ERROR_REPORTING_URLURL to send sanitized error reports to

Encrypted Environment Variables

Connection strings and other sensitive values can be encrypted using gate vault so they are never stored in plaintext:

# Encrypt your production secrets
gate vault keygen
gate vault encrypt "postgresql://user:pass@host/db"
# Use the VAULT: prefix in your env
DATABASE_URL=VAULT:eyJhbGdv...
REDIS_URL=VAULT:eyJhbGdv...

The vault key must be present on the production server at ~/.gate/vault.key (chmod 0600). Variables prefixed with VAULT: are automatically decrypted at server startup. See Vault & Encryption for full details.

Deploy Steps

# 1. Build
npm run build

# 2. Set environment variables
export DATABASE_URL="postgresql://user:pass@host:5432/gate_prod"
export REDIS_URL="redis://host:6379"
export JWT_SECRET="your-production-secret-at-least-32-chars"
export ENCRYPTION_KEY="64-hex-character-string-for-aes256"
export SESSION_SECRET="your-session-secret-at-least-32-chars"
export GATE_LICENSE_SECRET="your-license-signing-secret"
export GITHUB_CLIENT_ID="your-github-oauth-app-id"
export GITHUB_CLIENT_SECRET="your-github-oauth-app-secret"
export GITHUB_CALLBACK_URL="https://your-domain.com/auth/github/callback"
export DASHBOARD_URL="https://your-domain.com"
export NODE_ENV="production"

# 3. Run database migrations
npx prisma db push --schema prisma/schema.prisma

# 4. Start the server
node dist/backend/server.js

# 5. Start workers (separate process)
node dist/workers/index.js

Docker

FROM node:20-slim
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci --production
COPY dist/ dist/
COPY bin/ bin/
COPY src/cli/ src/cli/
COPY rules/ rules/
COPY prisma/ prisma/
RUN npx prisma generate --schema prisma/schema.prisma
EXPOSE 3000
CMD ["node", "dist/backend/server.js"]

Process Management

In production, run two processes:

  1. gate serve (or node dist/backend/server.js) — API + dashboard
  2. gate worker (or node dist/workers/index.js) — background jobs

Use PM2, systemd, Docker Compose, or your platform's process manager to keep them running.

Troubleshooting

"Database does not exist"

createdb gate_prod
DATABASE_URL="..." npx prisma db push --schema prisma/schema.prisma

"Redis connection refused"

Workers need Redis running. The API server works without Redis — skip workers if you don't need background jobs.

"Environment validation failed"

Set every required variable from the table above. In development, all have safe defaults.

"Frontend shows blank page after build"

npm run build:frontend
ls dist/frontend/index.html
