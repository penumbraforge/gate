GitHub Action

Server-side enforcement that can't be bypassed.

The Gate GitHub Action runs on every PR and push, scanning for secrets and blocking unsafe commits. Published separately to the GitHub Marketplace.

Basic Setup

# .github/workflows/gate.yml
name: Gate Security
on: [pull_request, push]
jobs:
  gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: penumbra/gate@v1
        with:
          mode: enforce
          failure-mode: block
          slack-webhook: ${{ secrets.SLACK_WEBHOOK }}

Inputs

InputDefaultDescription
modereportenforce blocks on findings, report just logs
failure-modeblockblock fails CI, warn passes but alerts
slack-webhook—Slack webhook URL for notifications
rules-versionlatestPin to specific rules version
github-token${{ github.token }}For PR comments

Outputs

OutputDescription
findings-countNumber of findings detected
blockedWhether the commit was blocked
scan-reportFull JSON scan report

How It Works

  1. Checks out the repository
  2. Runs the Gate scanner on changed files
  3. Reports findings as PR comments
  4. Can block merging (failure-mode: block) or just warn (failure-mode: warn)
  5. Optionally sends Slack notifications

Examples

Report Mode (Non-blocking)

- uses: penumbra/gate@v1
  with:
    mode: report
    failure-mode: warn

Findings are posted as PR comments but CI stays green.

Enforce Mode with Slack

- uses: penumbra/gate@v1
  with:
    mode: enforce
    failure-mode: block
    slack-webhook: ${{ secrets.SLACK_WEBHOOK }}

Findings block the PR and alert your security team via Slack.

Pinned Rules Version

- uses: penumbra/gate@v1
  with:
    mode: enforce
    rules-version: "2.1.0"

Why server-side? Developers can skip the pre-commit hook with git commit --no-verify. The GitHub Action is the enforcement layer that can't be bypassed.
